# 灵数仙途 ⚔️ Math Cultivation

> 面向小学生（6-12岁）的沉浸式修仙题材数学训练 RPG。
> **数道即仙道**——答题即施法，答得越快剑气越强，一路修炼从炼气直至真仙。

前作为 [math-speed-trainer（数学魔法学院）](https://github.com/szkailorik/math-speed-trainer)。本作全面重做：手绘分层 SVG 动漫立绘替代 Emoji、六大主题秘境视差场景、Canvas 粒子打击感、程序化五声音阶配乐，并继承强化了前作验证过的自适应教学引擎。

## 🎮 玩法

**核心循环**：进秘境 → 遇妖兽 → **答题即施法** → 胜利结算 → 修为突破

- ✅ **答对** = 剑气命中，伤害 ∝ 答题速度（疾如闪电 S / 快剑 A / 稳健 B / 迟缓 C）
- ❌ **答错** = 妖兽反击破护盾，立即显示正解 + 策略口诀，本局该题必重现
- 🔥 **五阶连击**：凝神 → 引灵 → 驭剑 → 剑心 → 天人合一（伤害 1.0x → 3.0x，主角形态光环递进）
- ⚡ **境界体系**：炼气 → 筑基 → 金丹 → 元婴 → 化神 → 炼虚 → 合体 → 大乘 → 渡劫 → 真仙，突破时有雷劫大演出
- 🐉 **灵兽收集**：18 只灵兽（灵品 R / 仙品 SR / 神品 SSR），妖王战与全三星达成时收服
- 👿 **心魔殿**：每道错题凝成心魔，同一题独立答对两次才能"洗白"
- 🌅 **每日晨课**：限时十题混合挑战 + 连续打卡
- 🤝 **双修共战（双人协作联机）**：两位小朋友两台设备，一人"开坛"生成 4 位灵咒，另一人输入加入。**各自独立答题，剑气共击同一只妖兽，同守一面队伍护盾**——不是对战，是并肩作战。伙伴的立绘、连击、伤害实时出现在你的画面里；一人看错题讲解时另一人可继续输出；伙伴掉线自动转单人继续，胜利双方各得"双修加成"修为

## 📚 训练内容（六大秘境）

| 秘境 | 模块 | 内容 |
|------|------|------|
| 🎋 青竹林 | 小九九 | 1~9 乘法表（重点打磨 6×7~8×9 高错率区） |
| 🏜️ 石纹漠 | 大九九 | 11~19 × 11~19 |
| ☁️ 流云涧 | 速算诀 | 25×4 / 125×8 凑整、平方数、两三位数加减 |
| ⚙️ 千机阁 | 度量术 | 长度 / 质量 / 时间 / 面积单位换算 |
| 🌊 碧波潭 | 小数诀 | 小数 ×÷10/100/1000、分数小数互换、比大小 |
| ✨ 星辰海 | 分数章 | 约分、同分母/同分子比较、同分母加减 |

每秘境 6 章（第 3、6 章为妖王战），星级评价（3星 = 90% 正确率且护盾无损）。

## 🧠 教学引擎（基于学习科学）

- **85% 法则**（Wilson et al., 2019）：隐藏动态难度把实时正确率钉在最优学习区
- **Leitner 熟练度模型**：每个知识点独立追踪（盒等级 + 响应时间双信号），答对 <1.5s 视为"自动化"
- **错题洗白机制**：答错的题需两次独立答对才恢复，且当局必重现（hypercorrection 效应）
- **连错保护**：连错 2 题自动插入已掌握的简单题"回血"+ 加时
- **郊狼时间**：倒计时归零后仍有 400ms 宽限，消解输入挫败
- **正向计时**：时间条是"聚灵进度"（越快伤害越高），概念题时限自动放宽

## 🎨 技术

纯静态、零依赖、零构建、零外部素材：

- **HTML5 + CSS3 + Vanilla JS (ES6+)**，全部美术为运行时生成的分层 SVG
- 主角伪 Live2D：呼吸 / 眨眼 / 发丝飘动 / 袖摆微动 / 连击光环，纯 CSS keyframes
- 28 种妖兽 + 18 只灵兽：参数化 SVG 工厂（HSL 调色 + 组件复用）
- 打击感：hit-stop 冻结帧、衰减屏震、伤害飘字、Canvas 对象池粒子（命中迸发/星尘/金粒喷泉）
- 六境视差背景 + 环境粒子（竹叶 / 流沙 / 云絮 / 齿轮 / 水泡 / 星屑）
- Web Audio 程序化音频：五声音阶 BGM（宗门/战斗双情绪）+ 合成音效（剑气/暴击/雷劫）
- localStorage 存档，PWA 可安装（iPad 优先适配），尊重 `prefers-reduced-motion`
- **联机**：WebRTC P2P（[Trystero](https://github.com/dmotz/trystero) @ Nostr 多中继信令，已 vendor 进仓库，无自建服务器）。同一 WiFi 下两台设备数据直连本地网络（延迟毫秒级），跨网走 STUN 打洞；对称模拟架构（双方各自应用全部伤害/护盾事件），战斗中带 Wake Lock 防锁屏断联
