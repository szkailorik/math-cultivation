# 《灵数仙途》设计文档 v1.0

> 面向小学生（6-12岁）的沉浸式修仙题材数学训练 RPG。纯静态网页（无构建工具），iPad 优先，localStorage 存档。
> 前作：math-speed-trainer（数学魔法学院）。本作全面升级美术拟真度与沉浸感，继承并强化其已验证的教学引擎。

## 一、世界观

你是灵峰宗最年轻的弟子。上古之时，"数道"乃修仙之本——九九为基，分数为脉，度量为骨。如今心魔妖兽侵蚀六大秘境，唯有以数道真诀斩妖除魔，方能步步突破，登临大乘。

**境界体系**（修为 = 经验值，境界突破有渡劫大演出）：
炼气 → 筑基 → 金丹 → 元婴 → 化神 → 炼虚 → 合体 → 大乘 → 渡劫 → 真仙（每境界 3 层：初期/中期/后期）

## 二、六大秘境（训练模块）

| 秘境 | 模块 | 内容 | 主色调 | 答题方式 |
|------|------|------|--------|---------|
| 青竹林 | 小九九 | 1-9 乘法表（重点高错率 6×7~8×9） | 竹青 | 数字键盘 |
| 石纹漠 | 大九九 | 11-19 × 11-19 | 沙金 | 数字键盘 |
| 流云涧 | 速算诀 | 25×4/125×8 凑整、两位数加减、平方 | 云白蓝 | 数字键盘 |
| 千机阁 | 度量术 | 长度/质量/时间/面积单位换算 | 铜赤 | 四选一 |
| 碧波潭 | 小数诀 | ×÷10/100、小数分数互换、比大小 | 水碧 | 混合 |
| 星辰海 | 分数章 | 约分/通分/比较/同分母加减 | 星紫 | 混合 |

每个秘境 6 章，每章一场战斗（4 只妖兽 + 第 3/6 章有 BOSS）。星级：3星=90%正确率且未阵亡，2星=75%，1星=通关。

## 三、核心循环

进秘境 → 遇妖兽 → **答题即施法**：
- 答对 → 剑气命中，伤害 ∝ 答题速度（S/A/B/C 判定，S=暴击）
- 答错 → 妖兽反击破护盾，**立即显示正解 + 一句策略口诀**，该题本局稍后必重现
- 连击 5 阶（凝神→引灵→驭剑→剑心→天人合一），伤害倍率 1.0→3.0，形态特效递进
- 胜利 → 灵石 + 修为 + 掉落；修为满 → **渡劫突破演出**（雷云、闪电、全屏粒子）
- BOSS 战胜利 → 有概率**收服灵兽**（图鉴 18 只，N/R/SR/SSR 稀有度）

单局 3-6 分钟收口。

## 四、教学引擎（核心资产，继承前作+调研强化）

1. **85% 法则**：实时正确率钉在 85% 附近。连对悄悄缩短时限/提升题目难度层，正确率 <75% 时插入已掌握题"回血"，绝不让孩子连错 3 题。
2. **Leitner 熟练度模型**：每个 fact（如 `m7x8`）记录 box 0-4、平均响应时间。<1.5s 答对=自动化（box+1 且降低复现频率），>3s 答对=会算但未自动化。
3. **错题洗白**：答错的 fact 需**两次独立答对**才恢复；错题在心魔殿（错题本）可专项挑战。
4. **郊狼时间**：倒计时归零后仍有 400ms 宽限。
5. **计时正向包装**：时间条是"聚灵进度"（越快灵气越足伤害越高），不是威胁性红字倒计时；概念题（单位/分数）时限放宽 1.4 倍。
6. **连错保护**：连错 2 题 → 下一题必出已掌握的简单题 + 时限 +2s + 鼓励语。
7. **hypercorrection**：答错立即给正解+策略提示（如 17×3 → 想 10×3 + 7×3）。

## 五、美术方案（对准前作最大短板）

- **主角**：分层手写 SVG 动漫立绘（后发/身体/袍/脸/五官/前发/剑），呼吸 scaleY 循环、随机眨眼、发丝飘动、连击阶段光环变化（伪 Live2D）。
- **妖兽**：SVG 工厂 + 手工 BOSS，有 idle 浮动/受击 squash&stretch/死亡消散动画，眼睛会随状态变化。
- **场景**：六秘境各自 3-4 层视差 SVG 背景 + 环境粒子（竹叶/流沙/云絮/齿轮光/水泡/星屑）。
- **打击感**：hit-stop 70ms、衰减屏震、伤害飘字（暴击金色大字）、Canvas 对象池粒子（命中迸发/连击星尘/升级金粒喷泉）、SVG 辉光滤镜、符咒 stroke-dashoffset 咏唱动画。
- **音频**：Web Audio 程序化五声音阶 BGM（宗门恬静/战斗紧凑）+ 合成音效（剑气/命中/暴击/受伤/突破雷鸣）。
- 全程尊重 `prefers-reduced-motion`。

## 八、双修共战（v1.1 双人协作联机）

**目标**：两个孩子两台设备共同打一只妖兽（协作，非对战）。

- **联网方案**（2026-07 调研结论）：Trystero 0.25.2（Nostr 多中继信令 + WebRTC DataChannel），无自建服务器、无单点；备选 PeerJS 官方云（口碑有抖动）。同 WiFi 下 P2P 数据走本地网络；iPad Safari DataChannel 成熟，纯数据通道无权限弹窗。
- **配对**：主机"开坛"生成 4 位灵咒（房号），客机输入加入。appId + password 隔离房间。
- **架构**：对称模拟——伤害/护盾事件双向广播，双方各自应用全部事件，状态收敛一致；无需主机权威仲裁（低频消息 + 求和语义）。妖兽阵容由主机在 start 消息里下发。
- **协作机制**：各自独立题目流（各自的 Adaptive 会话照常记录学习数据）；共享妖兽血量（加厚：小妖 6-7，妖王 12）；共享队伍护盾 6 点；答错看讲解卡不阻塞伙伴，8 秒自动继续；伙伴掉线转单人继续。
- **奖励**：胜利双方各得修为（+20% 双修加成）与灵石；不掉灵兽（保留单人价值）；结算显示"你的剑气"（个人输出）。
- **工程注意**：Trystero 0.25 API 为 `makeAction(name,{onMessage}) → {send}`、`room.onPeerJoin = cb`（赋值式回调）；战斗中申请 Wake Lock 防 iPad 锁屏断联；vendor 文件建议每半年更新一次以获得新鲜 relay 列表。

## 九、v1.2 优化（2026-07-06）

1. **六大秘境全开放**：移除境界解锁门槛（继承前作 math-speed-trainer 的开放模块制）——孩子想练哪个模块直接进；秘境内章节递进保留。
2. **主角立绘重绘**：发冠+发簪、符文仙剑（剑穗）、交领汉服金边、玉佩流苏、垂落鬓发、径向渐变瞳孔；新增飘带摆动/鬓发轻摇/足下灵气涡动画层；战斗新增剑气月牙特效（FX.slashArc，答对蓝月牙/会心金月牙飞向妖兽，答错红爪气反击）。
3. **答题输入方式（依据专项调研定稿）**：
   - 原则：**答案是"从记忆里取出一个数"→键盘输入；答案是"在几个对象间做判断"→选择题**
   - 证据：constructed response 优于 recognition（检索练习研究）；MC 有负面暗示效应（Roediger & Marsh 2005）与排除法漏洞；Reflex/XtraMath/Rocket Math 等 fluency 专业产品全部强制键盘输入；Prodigy 因选择题"猜答案"挨批
   - 落地：小九九/大九九/速算=键盘（不变）；**单位换算数值题改键盘**（答案≤5位，另 30% 出跨单位比大小的诊断性选择题）；**小数移位计算与分数→小数改带小数点键盘**（比大小/小数→分数保持选择）；分数模块维持选择题为主（正是研究推荐的辨析场景）
   - 键盘题按答案位数补偿限时（+300ms/字符，Reflex 的打字耗时扣除思路）；答案数值归一化比对（0.5==.5）
   - 不提供孩子可见的模式切换（Prodigy 教训：游戏激励下孩子会系统性选省力路径）
4. **资源加版本参数**（?v=N）：解决浏览器/GitHub Pages 的缓存更新延迟，每次发版需同步 bump index.html 的 ?v= 与 sw.js 的 CACHE 版本。
